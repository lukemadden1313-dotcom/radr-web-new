# Radr iMessage Share Cards — Web Implementation Spec

## Background

When a user shares a Radr workout, group, or profile via iMessage, the rich preview a recipient sees is generated by Apple's Link Presentation framework from Open Graph meta tags on the corresponding web page at `getradr.app`. Currently:

- `og:title`, `og:description`, `og:url` are dynamic per route ✅
- `og:image` is the host's Supabase avatar — **this is the gap.** It should be a designed share card matching Eli's `spec-v4` (cobalt accent for workouts, green for groups, purple for profiles).

This spec covers two deliverables per route:

1. **OG image generation** — a 1200×630 PNG generated server-side from workout/group/profile data, served at a stable URL, set as `og:image`.
2. **Web fallback page** — what loads in Safari when a non-user taps the iMessage preview. Should match the same visual language as the OG card.

## Tech stack assumptions

- Next.js App Router (already in use — `_next/static/chunks` confirmed in HTML)
- Server components allowed
- Supabase JS client with anon key for public reads
- `next/og` (Next 13.3+) or `@vercel/og` for image generation

## Routes

| Route | Type | Accent | Existing? |
|---|---|---|---|
| `/w/[id]` | Workout | `#0C5DE9` cobalt | Yes — needs OG image upgrade |
| `/g/[id]` | Group | `#2AD472` green | Verify exists |
| `/u/[username]` | Profile | `#9A5AF0` purple | Yes — needs OG image upgrade |

For each route, generate the OG image at:

- `/w/[id]/opengraph-image.png` (Next.js convention)
- `/g/[id]/opengraph-image.png`
- `/u/[username]/opengraph-image.png`

Next.js will automatically wire these into the parent route's `<head>` — no manual `og:image` meta tag needed if you use the file-based convention.

## Card 1 — Workout `/w/[id]`

### Data requirements (from Supabase)

```sql
-- Single query joining workouts, profiles (host), and workout_participants
SELECT
  w.id,
  w.title,
  w.start_time,
  w.location,
  w.activity_name,
  w.category,
  w.deleted_at,
  w.hidden_at,
  host.id          AS host_id,
  host.username    AS host_username,
  host.full_name   AS host_full_name,
  host.avatar_url  AS host_avatar_url,
  (SELECT COUNT(*) FROM workout_participants wp
     WHERE wp.workout_id = w.id AND wp.status = 'going')                AS going_count,
  (SELECT json_agg(json_build_object(
            'username',   p.username,
            'full_name',  p.full_name,
            'avatar_url', p.avatar_url))
     FROM workout_participants wp
     JOIN profiles p ON p.id = wp.user_id
     WHERE wp.workout_id = w.id AND wp.status = 'going'
     ORDER BY wp.joined_at ASC LIMIT 5)                                  AS going_avatars
FROM workouts w
JOIN profiles host ON host.id = w.creator_id
WHERE w.id = $1;
```

If `deleted_at IS NOT NULL OR hidden_at IS NOT NULL` → render a "Workout unavailable" card with generic Radr branding, no host info.

### Helper: Avatar component

```tsx
function Avatar({ url, fallback, size }: { url?: string | null; fallback: string; size: number }) {
  if (url) {
    return (
      <img
        src={url}
        width={size}
        height={size}
        style={{ borderRadius: '50%', objectFit: 'cover' }}
      />
    );
  }
  // Gradient fallback with first letter
  const idx = fallback.charCodeAt(0) % radr.avatarGradients.length;
  const [from, to] = radr.avatarGradients[idx];
  return (
    <div style={{
      width: size,
      height: size,
      borderRadius: '50%',
      background: `linear-gradient(135deg, ${from}, ${to})`,
      display: 'flex',
      alignItems: 'center',
      justifyContent: 'center',
      color: '#fff',
      fontSize: size * 0.4,
      fontWeight: 600,
    }}>
      {fallback.charAt(0).toUpperCase()}
    </div>
  );
}

function AvatarStack({ avatars, max = 4 }: { avatars: any[]; max?: number }) {
  const visible = avatars.slice(0, max);
  return (
    <div style={{ display: 'flex' }}>
      {visible.map((a, i) => (
        <div key={i} style={{ marginLeft: i === 0 ? 0 : -14, border: '4px solid #000', borderRadius: '50%' }}>
          <Avatar url={a.avatar_url} fallback={a.full_name || a.username || '?'} size={48} />
        </div>
      ))}
      {avatars.length > max && (
        <div style={{
          marginLeft: -14,
          width: 48,
          height: 48,
          border: '4px solid #000',
          borderRadius: '50%',
          background: '#1a1a1a',
          color: '#fff',
          fontSize: 18,
          fontWeight: 700,
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}>
          +{avatars.length - max}
        </div>
      )}
    </div>
  );
}
```

### Helper: utility functions

```tsx
function truncate(s: string, max: number): string {
  if (!s) return '';
  return s.length > max ? s.slice(0, max - 1).trimEnd() + '…' : s;
}

function formatDayTime(iso: string): string {
  // "WED · APR 29 · 7:00 PM" — uppercase, dot-separated.
  // Render in Eastern Time as a default; switch to per-workout timezone if
  // you store one on the workouts table.
  const d = new Date(iso);
  const tz = 'America/New_York';
  const day = d.toLocaleString('en-US', { weekday: 'short', timeZone: tz }).toUpperCase();
  const month = d.toLocaleString('en-US', { month: 'short', timeZone: tz }).toUpperCase();
  const date = d.toLocaleString('en-US', { day: 'numeric', timeZone: tz });
  const time = d.toLocaleString('en-US', { hour: 'numeric', minute: '2-digit', hour12: true, timeZone: tz });
  return `${day} · ${month} ${date} · ${time}`;
}

          background: 'linear-gradient(180deg, #0d0d0d 0%, #060606 100%)',

          <div style={{ display: 'flex', justifyContent: 'space-between', alignItems: 'center' }}>
            <span style={{ fontSize: 28, fontWeight: 700, fontStyle: 'italic', color: '#FFFFF7' }}>
              radr<span style={{ color: '#666' }}>.</span>

              padding: '8px 18px',

            {label}
          </span>
          <span style={{ fontSize: 24, color: '#888' }}>
            This may have been removed or hidden.
          </span>
        </div>
      </div>
    ),
    { ...size, fonts: await loadFonts() }
  );
}
```

## Card 3 — Profile `/u/[username]`

### Data requirements

```sql
SELECT
  p.id,
  p.username,
  p.full_name,
  p.avatar_url,
  p.bio,
  p.total_workouts,
  p.current_streak,
  (SELECT COUNT(*) FROM friendships f
     WHERE f.status = 'accepted'
       AND (f.requester_id = p.id OR f.receiver_id = p.id))           AS friend_count
FROM profiles p
WHERE LOWER(p.username) = LOWER($1);
```

## Shared helper: `loadFonts()`

This was referenced in all three card files but never defined. Drop into `app/og-helpers.tsx` (or wherever you consolidate the OG helpers) and import where needed.

```tsx
// Loads Inter at 400/600/800 weights for use inside next/og's edge runtime.
// Edge runtime can't use local fs, so fonts must be fetched at request time
// (or, ideally, bundled and read via dynamic import — see "Production
// optimization" note below).
export async function loadFonts() {
  const [regular, semibold, bold] = await Promise.all([
    fetch('https://github.com/rsms/inter/raw/master/docs/font-files/Inter-Regular.otf').then(r => r.arrayBuffer()),
    fetch('https://github.com/rsms/inter/raw/master/docs/font-files/Inter-SemiBold.otf').then(r => r.arrayBuffer()),
    fetch('https://github.com/rsms/inter/raw/master/docs/font-files/Inter-ExtraBold.otf').then(r => r.arrayBuffer()),
  ]);
  return [
    { name: 'Inter', data: regular,  weight: 400 as const, style: 'normal' as const },
    { name: 'Inter', data: semibold, weight: 600 as const, style: 'normal' as const },
    { name: 'Inter', data: bold,     weight: 800 as const, style: 'normal' as const },
  ];
}
```

**Production optimization (recommended):** fetching fonts on every cold start adds ~200–400ms to the OG render. Bundle the three `.otf` files under `app/fonts/` and replace the fetches with:

```tsx
const regular  = await import('./fonts/Inter-Regular.otf?arraybuffer');
const semibold = await import('./fonts/Inter-SemiBold.otf?arraybuffer');
const bold     = await import('./fonts/Inter-ExtraBold.otf?arraybuffer');
```

(Requires Next.js `experimental.fontLoaders` or a Webpack arraybuffer loader — verify with the version of Next.js in use.)

## Caching strategy

Add to every `opengraph-image.tsx`:

```tsx
export const revalidate = 300; // 5 minutes
```

5-minute edge revalidation balances freshness vs. cost:

- Workout edits, RSVP changes, new comments propagate within 5 min
- Each card render hits Supabase once per 5 min per route, not per request
- iMessage caches preview client-side per URL — to force fresh during testing, append a query string: `https://getradr.app/w/<id>?v=2`

For the fallback HTML page (`page.tsx`), use the same `revalidate = 300` unless live participant data needs to be more current.

**Cache headers** on the OG image response are set automatically by `next/og`. If you ever serve the image through a custom API route instead, ensure the response includes:

```
Cache-Control: public, max-age=300, s-maxage=300, stale-while-revalidate=86400
Content-Type: image/png
```

## Error states

For each route, when data is missing/deleted/hidden, render the `notFoundCard()` helper:

- `/w/<id>` — workout has `deleted_at IS NOT NULL` or `hidden_at IS NOT NULL`, or no row matches the ID
- `/g/<id>` — group doesn't exist or has been deleted
- `/u/<username>` — username doesn't match any profile (case-insensitive)

The page should still return **HTTP 200** and render the gray "Unavailable" card. Don't 404-redirect — Apple won't render an iMessage preview for non-200 responses, which breaks the link unfurl chain.

For the HTML fallback `page.tsx`, render a similar "no longer available" message with a single CTA to the App Store. Same HTTP 200 rule applies for consistency.

## Testing checklist

For each card type (workout, group, profile):

1. **OG tags present and dynamic:**

   ```
   curl -sS -L -A "Mozilla/5.0" "https://getradr.app/w/<real-id>" | grep -E "og:|twitter:"
   ```

   Verify all 5 tags resolve to real data, not generic Radr brand (the regression we hit before).

2. **OG image renders correctly:**
   Open `https://getradr.app/w/<real-id>/opengraph-image.png` directly in browser. Verify 1200×630, all spec elements present, no clipping, avatar fallback (gradient + first letter) renders when `avatar_url` is null or fails to load.

3. **Facebook Sharing Debugger** — forces re-scrape, shows what every OG consumer including iMessage sees:
   `https://developers.facebook.com/tools/debug/?q=https://getradr.app/w/<id>`
   Click "Scrape Again" to bust their cache.

4. **iMessage live test:** From a Mac with Messages, send the URL to yourself. Use a brand-new conversation thread (or append `?v=test`) to bypass cache. Preview card should match the OG image exactly.

5. **Universal Link handoff:** Tap the iMessage preview on an iPhone with Radr installed. Should deep-link into the app's workout / group / profile detail view. (Already working — verify still works after any URL changes.)

6. **Web fallback (no Radr installed):** Tap on a device without Radr. Safari should load the styled page matching the same accent. Both CTAs should work — deep link opens app if installed, App Store link opens store.

7. **Error state:**
   Hit `/w/00000000-0000-0000-0000-000000000000` (a UUID that doesn't exist). Should return HTTP 200 with the gray "Workout unavailable" card. iMessage preview should render this gracefully.

8. **Edge runtime cold start:**
   First request to a card route after deploy should complete within ~1.5s. If consistently >3s, the font fetch is likely the bottleneck — apply the bundled-font optimization in `loadFonts()`.

## Out of scope for this implementation

- iOS app changes (already ships correct share URLs via `Utilities/ShareURLs.swift` and uses `UIActivityViewController`)
- Email previews / Slack unfurls (same OG tags serve these adequately; iMessage is the visual priority)
- Animated cards (iMessage only renders static images)
- Video previews / link unfurls beyond static OG image
- Localization of card text (English only for v1)
- Custom font weights beyond 400/600/800 (use `font-style: italic` as a CSS-level effect; don't add `Inter-Italic.otf` unless the spec evolves to require it)
